How Pricing Works

The commercial structure is straightforward even without public tiers.

The current model is built around a one-year software term plus a scoped implementation motion. That keeps the proposal aligned to the actual systems, workflow complexity, and onboarding work needed to get the first phase live.

Software subscription

  • Quoted as a custom annual subscription
  • Initial term assumed to be 12 months
  • Commercial scope shaped around the intended deployment, not generic bundles
  • Best fit for teams running meaningful receivables workflow complexity already

Implementation and onboarding

  • Scoped separately from the annual software term
  • Includes onboarding, workflow review, and environment-specific setup
  • Integration work is planned during onboarding rather than sold as a self-serve add-on
  • Manual client onboarding is the default approach today
Commercial Drivers

What usually changes the quote.

Even without packages, the same practical factors tend to shape the proposal from one deployment to the next.

Receivables and invoice volume

The size of the working book and the operational load on the team affect both software scope and onboarding complexity.

Data environment

ERP structure, CRM context, planned billing sources, and reporting requirements all change how the rollout is shaped.

Workflow complexity

Dunning stages, dispute handling, shared ownership, and forecast review depth influence how much operating setup is required.

User footprint

The number of operators, managers, reviewers, and administrators matters more than a generic seat count headline.

Integration depth

A first phase built around core API connections and validated workflow scope prices differently than a broader multi-system rollout.

Support expectations

Buyer requirements around onboarding cadence, review sessions, and ongoing support can affect the final commercial structure.

FAQ

Common commercial questions.

Is pricing published by tier?

No. The site currently routes pricing through a direct conversation because packaging is still tied closely to deployment scope.

Is the first term annual?

Yes. The current assumption on the site is a 12-month initial term.

Is implementation included in the software fee?

No. The current structure treats implementation as a separately scoped workstream.

Can teams start small?

Yes. The practical path is usually to scope the first workflow phase cleanly rather than trying to price every future integration up front.
