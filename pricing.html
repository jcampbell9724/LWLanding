How Pricing Works

The commercial structure is straightforward even without public tiers.

The current model is built around a one-year software term plus a scoped implementation motion. That keeps the proposal aligned to the actual files, workflow complexity, and onboarding work needed to get the first phase live.

Software subscription

  • Quoted as a custom annual subscription
  • Initial term assumed to be 12 months
  • Commercial scope shaped around the intended deployment, not generic bundles
  • Best fit for teams already running meaningful receivables workflow complexity

Implementation and onboarding

  • Scoped separately from the annual software term
  • Includes onboarding, workflow review, and environment-specific setup
  • Data intake work is planned during onboarding rather than sold as a self-serve add-on
  • Managed client onboarding is the default approach today
Commercial Drivers

What usually changes the quote.

Even without packages, the same practical factors tend to shape the proposal from one deployment to the next.

Receivables and invoice volume

The size of the working book and the operational load on the team affect both software scope and onboarding complexity.

Data intake scope

Receivables file shape, planned billing inputs, reporting needs, and any customer-specific ingest work all influence the rollout.

Workflow complexity

Draft follow-up stages, review cadence, and forecast depth influence how much setup is required in the first phase.

User footprint

The number of operators, managers, and reviewers matters more than a generic seat-count headline.

Implementation depth

A first phase built around validated file intake prices differently than a broader rollout that also includes customer-specific API ingest.

Support expectations

Buyer requirements around onboarding cadence, review sessions, and ongoing support can affect the final commercial structure.

Pricing FAQ

Questions buyers usually ask before commercial review.

Is pricing published by tier?

No. Ledgewave currently prices through a direct conversation because packaging still depends on workflow scope, onboarding work, and the first rollout phase.

Is the first term annual?

Yes. The current commercial assumption is a 12-month initial software term.

Is implementation included in the software fee?

No. Implementation and onboarding are scoped separately from the software subscription.

Can teams start with a smaller first phase?

Yes. The practical path is usually to scope the first workflow phase cleanly rather than pricing every future enhancement up front.
